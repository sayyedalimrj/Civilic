# Civilic — جهت UX/UI

## سبک
- Persian enterprise SaaS: ساده، آرام، حرفه‌ای، قابل‌فهم در ۱۰ ثانیه.
- پس‌زمینه‌ی روشن، کارت‌های نرم، فاصله‌ی منظم، تایپوگرافی قوی، پالت رنگ محدود (سرمه‌ای/آبی نفتی + accent کهربایی/سبز محدود).
- light mode اولویت دارد؛ dark mode اختیاری.
- تمرکز desktop/tablet سازمانی، اما responsive.

## ناوبری اصلی (فقط همین‌ها)
کارتابل من · پروژه‌ها · پیام‌ها · اسناد و مکاتبات · گزارش‌ها و خروجی · تنظیمات

**در ناوبری اصلی نمایش نده:** داده خام تکسا، BI، heatmap، map، supplier، risk، radar، forecast، آمار fake، KPI تصادفی. ماژول‌های پیشرفته/دیباگ زیر «تنظیمات → پیشرفته».

## تب‌های صفحه‌ی پروژه (فقط ۷ تب)
پیشخوان پروژه · اسناد و مکاتبات · متره و صورتجلسات · صورت‌وضعیت‌ها · تعدیل · گفتگو · گزارش و خروجی

## قواعد هر صفحه
- یک هدف روشن، یک primary action.
- وضعیت‌های قابل‌فهم، empty state تمیز، loading/skeleton تمیز، پیام خطا/موفقیت فارسی.
- جدول‌های بزرگ: pagination/virtualization، فیلتر روشن، اسکرول مناسب RTL.
- نمایش تاریخ شمسی و اعداد فارسی در UI؛ داده‌ی عددی backend خراب نشود.
- بدون نام جدول خام تکسا برای کاربر عادی. به‌جای آن: متره، خلاصه متره، برگه مالی، صورت‌وضعیت، تعدیل، کسورات، حمل، مصالح، فهرست‌بها.

## کارتابل من (Dashboard)
بخش‌ها: اقدامات منتظر من · صورت‌وضعیت‌های منتظر بررسی · پیام‌های خوانده‌نشده · مکاتبات ارجاع‌شده · پروژه‌های اخیر · هشدارهای مهم · یک CTA اصلی نقش‌محور.

## RTL/Persian (الزامی برای Vercel preview)
- `html lang="fa" dir="rtl"`.
- ترجیح CSS logical properties: `margin-inline`, `padding-inline`, `inset-inline`، `text-start`/`text-end` به‌جای left/right.
- dropdown/popover/sheet در سمت درست RTL باز شوند (Radix dir).
- فونت: `Vazirmatn, IRANSansX, YekanBakh, Tahoma, Arial, sans-serif` — بدون فونت پولی/خصوصی جدید.
