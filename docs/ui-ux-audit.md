# ممیزی UI/UX Civilic (پیش از فاز پولیش دسکتاپ‌محور)

> هدف: شناسایی مشکلات بصری و تعیین اصلاح دقیق. عملی، نه پرحجم.

## یافته‌های کلی
- پالت رنگ فعلی **کهربایی/نارنجی** است و حس «دمو/گرم» می‌دهد، نه SaaS سازمانی. رنگ‌های `amber-600`/`amber-50` به‌صورت hard-code در sidebar/header/workbench پخش شده‌اند.
- کامپوننت‌های مشترک طراحی (status badge، metric card، empty state، page header، responsive table) وجود ندارند؛ هر صفحه استایل خود را تکرار می‌کند.
- چیدمان دسکتاپ از عرض صفحه هوشمندانه استفاده نمی‌کند (workbench با `max-w-5xl` محدود است → بلوک‌های خالی در ۱۹۲۰px).
- سایدبار کاربری: هویت کاربر بالا، اما بدون منوی کاربر/خروج پایین؛ رنگ active کهربایی؛ روی موبایل permanent می‌ماند (بدون drawer).
- جداول مدیریت ساده‌اند اما empty/loading state و حالت کارت موبایل ندارند.

## جزئیات و اصلاح

| ناحیه | مشکل فعلی | چرا بد است | اصلاح |
|---|---|---|---|
| پالت | کهربایی hard-coded | غیرحرفه‌ای، ناهماهنگ | primary → ایندیگو/آبی عمیق در tokens؛ accent سبز فقط برای موفقیت |
| سایدبار کاربری | active کهربایی، بدون منوی کاربر، permanent در موبایل | حس دمو، شکستن موبایل | استفاده از توکن primary، profile+logout پایین، `hidden md:flex` + Sheet موبایل |
| هدر | برند «متره‌یار»/گرادیان کهربایی، شلوغ | برند قدیمی | برند Civilic با رنگ primary، ارتفاع کوتاه |
| کارتابل | `max-w-5xl`، کارت‌های بزرگ، چیدمان ضعیف دسکتاپ | فضای خالی در دسکتاپ | چیدمان grid چندستونه، metric cards فشرده، CTA نقش‌محور |
| داشبورد مدیریت | کارت‌های inline، بدون ساختار | کم‌مایه | metric cards مشترک + جدول‌های recent + اقدام‌های سریع |
| فهرست پروژه | جدول/کارت خام | حس دمو | کارت‌های پروژه‌محور با نقش/وضعیت/اقدام ورود |
| جداول مدیریت | بدون empty/loading/کارت موبایل | ناقص | ResponsiveTable + EmptyState + Skeleton مشترک |
| وضعیت‌ها | badge پراکنده | ناهماهنگ | StatusBadge مشترک با نگاشت فارسی + رنگ استاندارد |
| RTL | dropdown/popover ممکن است در LTR تراز شوند | تراز اشتباه | DirectionProvider (در صورت موجودبودن) + dir روی Radix |
| تایپوگرافی | عناوین ضعیف | سلسله‌مراتب ضعیف | PageHeader مشترک با عنوان قوی + زیرعنوان |
| حالت‌ها | بدون empty/loading/error یکنواخت | حس ناقص | کامپوننت‌های EmptyState/Skeleton/خطای فارسی |

## اولویت اجرا (دسکتاپ‌محور)
1. توکن‌های طراحی + recolor به ایندیگو.
2. کامپوننت‌های مشترک: StatusBadge، MetricCard، EmptyState، PageHeader، SectionCard، ResponsiveTable.
3. سایدبار کاربری (دسکتاپ قوی + Sheet موبایل) و هدر.
4. بازطراحی کارتابل من.
5. بازطراحی داشبورد مدیریت + جداول مدیریت با کامپوننت‌های مشترک.
6. فهرست پروژه‌ها.
7. حالت‌های empty/loading/error.
